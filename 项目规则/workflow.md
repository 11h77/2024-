# 金融学大创项目总规则（AI必须严格遵守）
## 一、通用规则
1. 所有内容必须具体、有出处，禁止空泛论述
2. 禁止使用"综上所述""由此可见"等套话，直接说结论
3. 所有引用必须标注作者+年份，没有DOI的文献不能引用
4. 任何修改必须说明理由，重要决定写进MEMORY.md

## 二、文献与选题规则
1. 只搜索2010-2026年的CSSCI、SSCI、北大核心期刊论文
2. 优先引用《经济研究》《管理世界》《金融研究》《世界经济》
3. 每个选题至少覆盖30篇核心文献，近3年文献占比≥60%
4. 文献综述必须包含：研究脉络、主流观点、争议点、研究空白
5. 边际贡献必须具体到某一个点，禁止"丰富了相关研究"这种空话
# 三、文献引用强制格式（AI必须100%执行）
1. 正文所有文献引用：**带方括号的数字整体上标**，格式是上标的 `[1]`、`[2]`、`[3]`，不是普通 `[1]`，也不是单独的 `¹`
   标准示例：数字普惠金融显著缓解中小企业融资约束`[1]`（其中整个 `[1]` 必须设置为上标）
2. 标注紧跟在被引用的结论/观点后面，不能集中放在整段末尾，也不能单独放句末凑数
3. 文末参考文献严格按 GB/T7714 排版，必须带 DOI
   期刊格式：[1] 作者. 论文题目[J]. 期刊名, 年, 卷(期): 起止页码. DOI
4. 禁止普通括号、禁止无标注、禁止编造文献/DOI
5. 综述、边际贡献、选题说明全程强制此格式
6. 生成 Word 文档时，必须用 Word 的上标格式处理整个引用标号 `[数字]`；生成 Markdown 草稿时，用 `<sup>[数字]</sup>` 表示

## 四、禁止行为
1. 绝对禁止编造文献、编造数据、编造结果
2. 禁止把相关性说成因果性，所有因果必须说明识别策略
3. 禁止过度外推结论，必须明确研究的边界和局限性
4. 禁止删除极端值，必须用winsorize 1%/99%处理

## 五、awesome-ai-research-writing 技能使用规则
技能来源：D:\大创\skill\awesome-ai-research-writing-main\awesome-ai-research-writing-main\README.md

### 1. 这个技能能做什么
1. 它不是金融学文献数据库，而是一套学术写作流程和提示词库。
2. 它能帮助完成：文献综述写作、论文段落润色、逻辑检查、去AI味、审稿人视角检查、图表标题、实验分析、Word文档处理。
3. 它强调每次写作前先明确任务、约束、输出格式，写完后用审稿人视角检查逻辑漏洞、格式问题和空话。

### 2. 和本项目规则的优先级
1. 本文件第一、二、三部分的金融学大创规则优先级最高。
2. 如果 awesome 技能里的规则和本项目规则冲突，必须按本项目规则执行。
3. awesome 技能原本偏计算机论文，本项目使用时要改成金融学、财政学、企业金融语境，不能照搬计算机会议表达。

### 3. 我的固定工作流程
1. 先读需求：确认这次是选题、找文献、写综述、写边际贡献、改论文，还是做实证方案。
2. 再查文献：只找2010-2026年的CSSCI、SSCI、北大核心期刊论文，优先《经济研究》《管理世界》《金融研究》《世界经济》，每条文献必须核对作者、年份、期刊、DOI和来源。
3. 再筛文献：按研究对象、核心变量、方法、结论、局限分组，不能只堆摘要。
4. 生成选题：每个选题必须说清楚研究问题、解释变量、被解释变量、机制变量、可能数据、识别策略和能做出的一个具体边际贡献。
5. 写文献综述：必须包含研究脉络、主流观点、争议点、研究空白，不能只按文献一篇篇罗列。
6. 写边际贡献：必须具体到一个点，例如“把土地财政依赖度下降拆成过度投资和投资不足两个方向检验”，禁止写“丰富了相关研究”。
7. 最后自查：用审稿人视角检查有没有编造文献、有没有无DOI引用、有没有把相关说成因果、有没有空话和套话。

### 4. 写作风格要求
1. 先用大白话解释，再给正式论文表达。
2. 重要判断必须说明理由，不允许只给结论。
3. 论文正文要自然、克制、具体，避免“具有重要意义”“深刻影响”“丰富了研究”等空泛表达。
4. 修改文字时必须说明为什么改；重要决定要写进 D:\大创\MEMORY.md。

## 五、后续补充工作流
1. 先做“选题可行性筛选”，再写正式综述。筛选标准包括：能不能找到足够核心文献、变量能不能拿到、有没有清楚机制、有没有可行识别策略。
2. 文献分成两类：正式引用文献和待核验候选文献。正式引用文献必须有作者、年份、期刊、DOI和来源；待核验候选文献只能用于提醒继续查，不能写进正式综述正文。
3. 每个选题都要先画清楚“主效应链条”和“调节效应链条”，再决定是否继续做。
4. 每个选题都要给一个“能落地的数据表”，包括被解释变量、解释变量、调节变量、机制变量、控制变量、数据来源。
5. 边际贡献必须写成“我比已有文献多做了哪一个具体检验”，例如“把社会保障支出拆成社保就业支出和养老保障指数两类结果变量”，不能写成泛泛的理论贡献。
6. 因果表述必须分级：普通双向固定效应只能说“相关关系”；只有使用DID、工具变量、政策冲击、断点回归等识别策略并通过检验，才可以说“影响”或“因果效应”。
